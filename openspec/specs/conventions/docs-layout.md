# docs-layout 分工邊界

- `docs/research/`：探索與研究原始材料
- `docs/superpowers/specs/`：brainstorming 後的設計稿
- `docs/superpowers/plans/`：可執行計畫
- `docs/superpowers/workstreams/`：按功能分流的 plan/task/todo
- `docs/ops/`：運維手冊與復原流程
- `openspec/specs/`：canonical stage 規格基線
- `openspec/changes/`：每次變更的 proposal/design/tasks 與歸檔
